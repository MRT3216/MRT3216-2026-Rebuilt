# Controller Cheat Sheet — SIM Mode

This cheat-sheet summarizes the Xbox controller bindings used in SIM mode for the MRT3216 robot.

## Defaults (always active)

- Flywheel default: `flywheelSubsystem.setVelocity(RPM.of(0))` — keep flywheel commanded to 0 RPM.
- Kicker default: `kickerSubsystem.setVelocity(RPM.of(0))` — keep kicker stopped.
- Spindexer default: `spindexerSubsystem.setVelocity(RPM.of(0))` — keep spindexer stopped.
 - Hood default: `hoodSubsystem.setAngle(() -> hoodSubsystem.getPosition())` — hold current hood position.

## Buttons (SIM mappings)

- A (while held)
  - Action: Run flywheel at 1000 RPM (closed-loop).
  - Effect: `flywheelSubsystem.setVelocity(RPM.of(1000))` while held. Reverts to default on release.

- B (while held)
  - Action: Run flywheel at tuned shooting speed (3000 RPM).
  - Effect: `flywheelSubsystem.setVelocity(RPM.of(3000))` while held.

- X (while held)
  - Action: Run simplified shoot routine (`shooterSystem.shoot()`).
  - Effect: Spins flywheel, runs a short clear while spinning up, waits for ~98% speed, then runs kicker + spindexer to feed.

- Y (while held)
  - Action: Run kicker then spindexer.
  - Effect: `kickerSubsystem.setVelocity(RPM.of(2000)).withTimeout(1s).andThen(spindexerSubsystem.setVelocity(RPM.of(4000)))`.

- Left bumper (on press)
  - Action: Hood preset to 15°.
  - Effect: `hoodSubsystem.setAngle(Degrees.of(15))`.

- Right bumper (on press)
  - Action: Hood preset to 45°.
  - Effect: `hoodSubsystem.setAngle(Degrees.of(45))`.

- Left trigger (threshold 0.1) (while held)
  - Action: Decrease hood target angle while held (fine adjust).
  - Effect: Supplier-backed `setAngle` computing (current hood angle - leftTrigger * 2°).

- Right trigger (threshold 0.1) (while held)
  - Action: Increase hood target angle while held (fine adjust).
  - Effect: Supplier-backed `setAngle` computing (current hood angle + rightTrigger * 2°).

- START (while held)
  - Action: Run shooter clear routine (reverse/clear kicker + spindexer).
  - Effect: `shooterSystem.clear()` while held.

## Turret test controls

- BACK (while held)
  - Action: Manual turret steering using the left stick X axis.
  - Effect: While BACK is held, the left stick X adjusts the turret target angle incrementally (approx. 5° per loop at full left-stick deflection).

- Left-stick press (on press)
  - Action: Snap turret to -90°.
  - Effect: `turretSubsystem.setAngle(Degrees.of(-90))`.

- Right-stick press (on press)
  - Action: Snap turret to +90°.
  - Effect: `turretSubsystem.setAngle(Degrees.of(90))`.

## Notes

- Turret has no active button bindings in the current SIM setup.
- All velocities and angles are units-aware (use `Units.RPM.of(...)`, `Units.Degrees.of(...)`).
- `shooterSystem.shoot()` is a composed command: spin flywheel, run timed clear during spin-up, wait for speed, feed.

- Manual turret steering scale: the implementation uses a 5° per-loop scale factor. If you want finer or coarser control, change the multiplier in `RobotContainer`.
